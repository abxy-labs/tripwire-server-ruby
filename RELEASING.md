# Releasing `tripwire-server`

This repository uses a manual `release.yml` workflow. Versions are independent from the other Tripwire server SDKs.

## Before the first real release

Configure these GitHub and registry settings:

1. Make the repository public.
2. Create a protected GitHub environment named `release` with required reviewer approval.
3. Create or configure the `tripwire-server` gem on RubyGems.
4. Add a RubyGems trusted publisher for:
   - repository: `abxy-labs/tripwire-server-ruby`
   - workflow: `.github/workflows/release.yml`
   - environment: `release`

## Release flow

1. Bump `lib/tripwire/server/version.rb` to the target version in a pull request.
2. Merge the version bump to `main`.
3. Open GitHub Actions and run `Release`.
4. Set `confirm_version` to the exact Ruby version constant.
5. Run a `dry_run=true` release first.
6. Re-run with `dry_run=false` once the dry run looks correct and the `release` environment approval is in place.

## What the workflow does

- reads the release version from `lib/tripwire/server/version.rb`
- reruns the equivalent of the repo checks
- builds the `.gem` artifact
- on real releases:
  - creates tag `vX.Y.Z`
  - publishes `tripwire-server` to RubyGems with trusted publishing
  - creates a GitHub Release with the built `.gem` attached

GitHub Release notes use autogenerated notes. There is no prerelease channel in this phase.
